About us

Preperation

I started this project by thinking about the game and overall topic before giving any thought to the technical implementation. Doing so helped me create a good base foundation to work with. I wanted to focus on the aspect of healthy eating and educating children age 5-10 on what foods are healthy and which aren't. I came up with a simple game where kids are presented with two images showing one healthy and unhealthy food item. They need to choose which option is healthy. If they decide correctly, they receive 2 points, and if they choose incorrectly, they will be explained what the correct answer was. The first player to 10 points wins.

Design

After that, I focused on finding the visual style of my webpage. I wanted to keep the design relatively simple to not distract from the content & topic. I decided to pick Opensans as my headline font and Lato for all body text. Next up, I thought about colors. To keep the minimalistic feel, I use black and white as my primary colors. To match the topic, I picked green as my main primary color. I used it for highlights in the navigation menu, the footer, and other components. To explain the game more accessible, I created small illustrations for each step. This helps children understand the game quicker and adds a playful touch.

Coding

I started this project with a blank canvas. First, I created the folder structure. I made my 4 HTML files, one folder for my CSS and another for all assets, including fonts and images. Because of the size of this project, I did not see the reason to add a folder for my HTML files. I started with creating a navigation bar. This was relatively easy for me because I have done this several times in the past. From there, I continued with adding content to the homepage. To finish the homepage, I added the footer. After completing the primary components (navbar & footer), I applied them to my other pages. As well as adding more specific content to each page. For the overall HTML, I keep it very simple with only using main attributes. CSS-wise, I tried to keep my file as short as possible and challenge myself to find the most efficient solutions for my problems. Trying to reuse one code snipped more often instead of adding the same line multiple times. We were allowed to use flexbox this time, which made the entire process a lot easier and quicker. Applying flexbox attributes to all my pages/content helped me achieve a reasonably easy responsive design.

Reflection

While working on this Oblig, I realized how much I have learned in the last three months. I had worked with HTML & CSS before and had a good understanding of both. But now, I feel way more confident and can solve code issues on my own. I could remember the work I had done in the previous Obligs, which helped me a lot. It showed that the hard work had paid off. Realizing this made me feel very proud and happy.
